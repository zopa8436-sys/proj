
Итоговое задание по курсу «Объектно-ориентированное программирование с использованием языка C++»
С использованием STL разработать решение с консольными приложениями для решения следующих задач (одно решение с четырьмя приложениями). 
Используйте меню, управление цветом и положением курсора.
Постарайтесь свести к минимуму использование циклов – максимально используйте алгоритмы STL. В качестве контейнера используйте класс vector&lt;&gt; STL.
Задача 1 (обязательная, максимум 2 балла)
Разработайте класс для решения задачи. Использовать алгоритмы, как можно используйте циклы. 
Создать вектор, из п вещественных (одинарной точности) элементов (n от 12 до 30), выполнить следующие обработки последовательно, сохранить вектор в бинарном файле перед обработками, восстановить вектор из бинарного файла после всех обработок (меню не требуется):
Количество отрицательных элементов вектора
количество элементов вектора, не попадающих в интервал [a, b], введенный с клавиатуры
сумму элементов вектора, расположенных перед первым минимальным элементом
сумму элементов вектора, расположенных между первым минимальным и первым максимальным элементами
выбрать в другой вектор отрицательные элементы
выбрать в другой вектор элементы, не попадающих в интервал [a, b]. Интервал из пункта 2)
упорядочить элементы вектора по убыванию
упорядочить вектор по возрастанию модулей элементов
упорядочить вектор по правилу – элементы, не попадающие в интервал значений [a, b] в конец вектора, интервал – из пункта 1)

Задача 2 (обязательная, максимум 2 балла)
Разработайте класс для решения задачи. Дан текстовый файл на русском языке в кодировке CP-1251. В файле слова разделяются одним или несколькими пробелами и символами перевода строки. В файле могут быть пустые строки. 
Для файла, с использованием класса string и алгоритмов STL реализуйте обработки:
Сформируйте частотный словарь слов текста. Ключом является слово, значением – количество таких слов в тексте, деленное на общее количество слов в тексте.
Сформируйте частотный словарь букв текста. Ключом является буква (регистро-независимая), значением – количество вхождений этой буквы в тексте, деленное на общее количество букв в теексте.
Поменять местами в тексте каждые две соседние строки, измененный текст сохранить в файле swap.txt
Перевести все слова в тексте в формат: первая буква верхнего регистра, остальные буквы в нижнем регистре (Это Пример Такого Формата), сохранить текст в файле capitalize.txt 
Упорядочить строки текста по длине, сохранить текст в файл с именем orderByLen.txt
В каждой строке текста упорядочить слова по алфавиту, оставить между словами по одному пробелу, сохранить измененный текст в файле с именем orderliness.txt 

Задача 3 (обязательная, максимум 4 балла)
Написать приложения для учета заявок на авиабилеты (естественно только то, что задано в спецификации, не полноценное приложение). Каждая заявка содержит: числовой идентификатор, пункт назначения, номер рейса (например, PO-2212K), фамилию и инициалы пассажира (Иванов П.О., …), желаемую дату вылета (класс Date, который был предложен в течение курса). 
В качестве контейнерного класса для коллекции заявок используйте класс list&lt;T&gt;.
Реализуйте обработки для списка заявок:
Начальная инициализация списка не менее чем 12 заявками
Добавление заявки в список, заявку формировать фабричным методом 
Удаление заявок из списка по идентификатору
Отбор заявок по заданному номеру рейса
Отбор заявок по дате вылета
Отбор заявок по фамилии и инициалам пассажира
Упорядочить заявки по идентификатору
Упорядочить заявки по желаемой дате вылета
Упорядочить заявки по пункту назначения 
Запись коллекции в бинарный файл с записями постоянной длины
Чтение коллекции из бинарного файла с записями постоянной длины
Поменять местами в файле (не загружая в коллекцию):
Первую и последнюю записи
Первую запись с самой ранней датой и первую запись с самой поздней датой
В классе приложения вызывать эти методы, выполнять отображение результатов, параметры вводить с клавиатуры. 

Задача 4 (обязательная, максимум 4 балла)
Разработайте класс для хранения коллекции плательщиков за телефонные услуги Payers за заданный месяц и год. Класс содержит список плательщиков за телефонные услуги, месяц и год периода учета, номер городской автоматической телефонной станции (ГАТС) для которой ведется список. Формат номера ГАТС - строка «062-xxx», где ххх – трехзначный номер ГАТС в диапазоне 121, …, 333).
Один элемент списка (класс Payer, плательщик) включает: информацию о плательщике: числовой идентификатор, фамилия и инициалы плательщика, номер телефона, тариф за минуту разговора, скидка (в процентах), суммарное время разговоров за месяц в полных минутах, дату платежа (поле класса Date). Также должен быть метод, вычисляющий сумму платежа.
Реализовать для класса Payers методы:
Метод добавления плательщика в список (при этом инкрементировать числовой идентификатор для нового плательщика)
Метод удаления плательщика из списка по идентификатору
Метод изменения плательщика по идентификатору. Менять все поля кроме идентификатора 
Метод отбора плательщика по номеру телефона, найденные записи помещать в отдельную коллекцию
Метод отбора плательщика по фамилии и инициалам, найденные записи помещать в отдельную коллекцию
Метод отбора плательщиков по дате платежа, найденные записи помещать в отдельную коллекцию
Метод отбора плательщиков по диапазону суммы платежа, сортировка по номеру телефона
Метод вычисления полной стоимости платежей всего списка плательщиков
Метод сортировки списка плательщиков по идентификатору плательщика
Метод сортировки списка плательщиков по номеру телефона
Метод сортировки списка плательщиков по убыванию суммы к оплате
Метод сортировки списка плательщиков по убыванию продолжительности разговоров 
В классе приложения вызывать эти методы, выполнять отображение результатов, параметры вводить с клавиатуры. Поля для изменения формировать фабричным методом, вводить с клавиатуры не надо.
Начальное заполнение данными реализуйте при помощи фабричного метода (фабричных методов). Храните объект класса Payers в файле формата CSV, структуру файла разработайте самостоятельно. Предусмотрите пункты меню для записи и чтения файла.
